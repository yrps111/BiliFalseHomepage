# B站UP主视频存档工具 — 操作指南

> 为任意B站UP主生成单文件自包含HTML视频存档网站。

## 项目结构

```
教学版-B站视频爬取工具/
├── bilibili_scraper.py     ← 主爬虫（爬前100页 + 下载封面）
├── browser_fetch.js        ← 浏览器控制台脚本（爬101页以后）
├── generate_html.py        ← HTML生成器（仿B站空间页风格）
├── B站视频爬取研究.md       ← 技术研究文档（博客文章）
├── 操作指南.md             ← 本文件
└── README.md               ← 项目说明
```

```bash
pip install curl_cffi Pillow
```

## 第一步：爬取前100页

1. 打开 `bilibili_scraper.py`，修改配置：
   ```python
   TARGET_MID = 目标UP主的UID  # 例如 123456789
   ```

2. 运行：
   ```bash
   python bilibili_scraper.py
   ```

3. 脚本会依次用 3 种排序方式爬取（每种最多100页 × 30条 = 3000条）：
   - `pubdate` — 按发布时间
   - `click` — 按播放量
   - `stow` — 按收藏量

4. 产出文件：
   - `videos_{UID}.csv` — 去重后的视频数据
   - `cover_cache.json` — 封面图片 base64 缓存

**注意事项：**
- 每次请求间隔 2~3.5 秒，5000+视频需要约2~3小时
- 中断后重新运行会从断点继续
- 3种排序的并集覆盖大部分视频，但总数>3000时可能仍有遗漏

## 第二步：爬取101页以后（如有需要）

> **为什么？** B站对 `pn > 100` 的API请求返回412，Python脚本无法绕过。但在已登录的浏览器中手动请求不受此限制。

1. Chrome打开任意B站页面（需要已登录）
2. F12 → Console（控制台）
3. 修改 `browser_fetch.js`：
   ```javascript
   const MID = "目标UID";
   const START_PAGE = 101;
   const END_PAGE = ???;    // 总视频数 ÷ 30，向上取整
   ```
4. 粘贴运行，完成后自动下载JSON

## 第三步：合并数据（如有第二步）

```python
import json, csv

existing = {}
with open('videos_UID.csv', 'r', encoding='utf-8-sig') as f:
    rows = list(csv.DictReader(f))
    for r in rows:
        existing[r['bvid']] = r

with open('videos_page101_plus.json', 'r', encoding='utf-8') as f:
    extra = json.load(f)

for v in extra:
    if v['bvid'] not in existing:
        existing[v['bvid']] = v

fieldnames = rows[0].keys()
with open('videos_UID.csv', 'w', encoding='utf-8-sig', newline='') as f:
    w = csv.DictWriter(f, fieldnames=fieldnames)
    w.writeheader()
    w.writerows(existing.values())

print(f'合并后共 {len(existing)} 条视频')
```

## 关键技术要点

### 反爬绕过

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| HTTP 412 | TLS 指纹不匹配 | `curl_cffi impersonate="chrome131"` |
| HTTP 412 | pn > 100 | 浏览器控制台手动爬 |
| -352 | WBI 签名无效/过期 | 每15~20页刷新密钥 |
| -799 | 请求频率过高 | 2~3.5秒随机间隔 |

### 100页限制

- B站API硬限制：`pn > 100` → 412
- 每种排序最多 100 × 30 = 3000 条
- 解决：多排序并集 + 浏览器手动爬

### HTML特性

- 仿B站空间页设计（毛玻璃效果）
- 搜索、排序、分页
- 电脑端：点击跳转B站
- 手机端：点击弹出BV号供复制
